## 计算机系统基础

> + 计算机是一种处理信息的工具
> + 它能自动、高速、精确地对信息进行存储、传输和加工处理

### 计算机的发展

#### 早期计算工具的发展

+ 商朝已经有了晚辈的文字系统和文字计数系统（一二三四五六七八九十）
+ 算筹起源于周朝
+ 算盘起源于唐朝，流行于宋朝

#### 早期计算机的发展

> 计算机的发展经历了机械式计算机、机电式计算机和电子计算机3个阶段

+ 1625 英国人 威廉+奥雷特 发明能进行6位数加、减法的华东计算尺
+ 1642 法国数学家帕斯卡 齿轮传动装置 进行8位十进制计算的加法器
+ 1822 英国数学家巴贝奇制造出差分机
+ 1941 德国科学家朱斯 电器元件制造计算机，全自动继电器计算机

##### 现代计算机

+ 1939 美国 阿塔纳索夫、

  > 只能解线性方程，无存储器

  ###### 基本原则

  1. 以二进制的方式实现数学运算和逻辑运算，以保证运算精度
  2. 利用电子技术实现控制和运算，以保证运算速度
  3. 采用计算功能于存储功能相分离的结构，以简化计算机设计

+ 1945 冯诺依曼 存储程序式通用计算机

  **电子计算机发展的萌芽时期结束，开始了现代计算机的发展时期**

#### 现代计算机的发展

> 在现代计算机诞生后60年中，计算机所采用的基本电子元件经历了四个阶段
>
> 电子管、晶体管、中小规模集成电路、大规模和超大规模集成电路

| 阶段 | 时间      | 基本电子元件             | 技术特点                                                     |
| ---- | --------- | ------------------------ | ------------------------------------------------------------ |
| 1    | 1946-1953 | 电子管                   | 【穿孔卡片和磁鼓】，使用机器语言和汇编语言                   |
| 2    | 1954-1963 | 晶体管                   | 主存储器采用【磁芯存储器】、磁鼓和磁盘做辅助、高级语言开发，小型计算机大量生产 |
| 3    | 1964-1970 | 中小规模集成电路         | 【半导体逐步取代】大型化、集中式计算、远程终端，出现了操作系统和编译系统 |
| 4    | 1971-现在 | 大规模和超大规模集成电路 | 【半导体完全取代】超大型化、计算机化、嵌入式、图形用户界面   |

### 计算机的基本工作原理及结构

#### 计算机的基本工作原理

##### **五个基本部件**

> 现代是被中往往将运算器和控制器制作在一个集成电路的芯片中，这个芯片被称为CPU

+ 运算器

  > 算术逻辑单元
  >
  > 是计算机进行算术运算和逻辑运算的部件
  >
  > 在控制器的控制下，运算和从存储器中取出数据进行运算，然后将结果写回存储器中

+ 控制器

  > 控制器用来控制程序和数据的输入和输出，以及各个部件的协调运行

  + 程序计数器
  + 指令寄存器
  + 指令译码器
  + 其他控制单元

  > 根据程序计数器中的地址，从存储器中取出指令到指令寄存器中，经译码单元译码后，再由控制器发出一系列指令信号，送到有关硬件部位引起相应动作，完成指令所规定的动作

+ 存储器

  > 主要功能是存储运行中的程序和数据
  >
  > 指的就是内存单元
  >
  > 每个存储单元的编号北纬内存地址
  >
  > 存储期的基本操作就是数据的写入或读出

+ 输入设备

  + 将显示世界中的数据输入到计算机中，转换成计算机所熟悉的二进制码
  + 由用户对计算机进行控制操作

+ 输出设备

  + 将计算机处理的结果转换成用户熟悉的形式

##### 采用二进制的形式表示数据和指令

##### 存储程序

> 核心思想
>
> 程序是解决问题的指令序列

#### 指令和指令系统

##### 指令

>  指令是能被计算机识别并执行的二进制代码，它规定计算机能完成的某一种操作

+ 数量和类型由CPU所决定

###### 程序

> 系统内存用于存放程序和数据，由指令和数据组成

+ 指令

  > 在内存中有序的存放，什么时候执行哪一条由控制器所决定

+ 数据

  > 用户需要处理的信息

  + 用户的具体数据
  + 这个数据在内存系统中的地址

  

###### 操作码和操作数

+ 操作码

  > 操作码知名该指令要完成的操作数类型和性质，如取数，做加法或输出数据

  操作码的位数决定了机器操作指令的条数，如果操作码的位数为N，则机器指令的条数为2^N

##### 指令系统

> 一台计算机的所有指令的集合被称为该计算机的指令系统

**一定具有如下指令**

+ 数据传送指令

  > 将数据在内存与CPU之间进行传送

+ 数据处理指令

  > 对数据进行算术、逻辑或关系运算

+ 程序控制指令

  > 控制程序中指令的执行、如条件转移，无条件转移，调用子程序

+ 输入\输出指令

  > 来实现外部设备与主机之间的数据传输

##### 指令的执行

> 所有复杂的事务处理都可以简化成两种最基本的操作
>
> 二进制数据的传输和二进制数据操作
>
> 所以，任意程序指令的执行都主要由“取指令、指令译码、指令执行、结果写回”4种基本操作构成

+ 取指令

  CPU内部有一个指令寄存器他保存当前所处理指令的内存地址

  控制器从指令寄存器中取得指令在系统内存中的地址，然后通过数据总线将将指令传送到CPU内部的指令高速缓存

+ 指令译码

  将解释指令的类型和内容，判断这条指令的操作数，并将操作数从内存单元读入CPU内部的高速缓存中，译码实际上就越是将二进制指令代码翻译成特定的CPU电路微操作，然后由控制器传回ALU（整数运算单元）

+ 指令执行

  根据不同的操作对象，传入不同的操作单元

  + ALU

    整数运算

  + FPU

    浮点运算

  > 预算过程中如果需要数据，则先从高速缓存中获取，如果高速缓存没有的话，则从内存中获取

+ 结果写回

  结果写回是将执行单元的处理结果写回到高速缓存或内存单元中

#### 微机系统结构

##### 早期微机系统结构

​	ISA（工业标准机构）总线结构

​	所有的设备挂接在一条ISA总线上，这条总线由控制总线，数据总线和地址总线组成，这种结构的优点是设计简单，缺点是所有设备都是用一条总线，容易产生系统瓶颈

##### 当前的微机系统结构

​	采用以CPU为核心的控制中心结构

##### 1-3-5-7 结构

+ 一个CPU

  CPU处于系统结构的最顶层，下面的数据需要逐层传递传递到最上层

  + 总线越往下走，性能越低
  + CPU的更新迭代将引起南北桥的改变

+ 三大芯片

  + 北桥芯片 （存储器控制中心）

    > 功能少
    >
    > 担任数据传输重任，其好坏决定主板性能的高低

    + 内存与CPU的数据交换
    + 显示数据与CPU数据的交换

  + 南桥芯片 （I/O控制中心）

    > 连接低俗设备，接口越多，扩展性越强

    + 数据的上传和下送

  + BIOS芯片

    + 关系到硬件系统与软件系统的兼容性

+ 五大接口
  + SATA
  + ESATA （外部硬盘接口）
  + SIO （超级输入输入接口）
  + LAN（以太网接口）
  + HDA（音频接口）
+ 七大总线

#### 计算机系统的组成

#### CPU系统

+ CPU是整个计算机的控制中心
+ 按照规定的脉冲频率，频率越高，CPU工作速度越快

##### CPU的技术性能

+ 处理字长

  > CPU内部 运算一次处理二进制数据的位数
  >
  > 位数是向下兼容的

  + 32位
  + 64位

+ 线宽

  > 硅晶片上各元件的连接线宽度，以纳米为单位，越小越高级

+ 高速缓存

  > 1 ~ 10M

+ 工作频率

##### CPU技术的新发展

+ 多核CPU

  > 带来了更强的并行处理能力
  >
  > 单核的CPU会触发频率高强，带来巨大的发热和功耗
  >
  > 最大问题是软件问题

+ 64位CPU

  > 让CPU通用寄存器的数据宽度为64位，一次处理64位二进制数据
  >
  > 必须要有64位的操作系统和64位应用软件的支持
  >
  > 64位CPU可以运行在32位操作系统上
  >
  > 62位CPU也可以运行在64位操作系统上
